## Repository Overview

This is a 3D visualization repository that serves as a centralized homepage for visualizing 3D reconstructions using the viser framework. It loads precomputed predictions (from VGGT) and provides an interactive 3D viewer for point clouds and camera poses.

## Architecture

### Core Components

**viser.py** - Main visualization script
- `viser_wrapper()` - Core function that:
  - Loads prediction dictionaries containing:
    - `images`: (S, 3, H, W) input images
    - `world_points`: (S, H, W, 3) 3D point positions
    - `world_points_conf`: (S, H, W) point confidence scores
    - `depth`: (S, H, W, 1) depth maps
    - `depth_conf`: (S, H, W) depth confidence scores
    - `extrinsic`: (S, 3, 4) camera extrinsics
    - `intrinsic`: (S, 3, 3) camera intrinsics
  - Creates interactive viser server with GUI controls
  - Supports confidence filtering, frame selection, and camera visualization
  - Can apply sky segmentation to filter out sky points
  - Supports two rendering modes: point map vs depth-based unprojection
  - Recenters point cloud around scene center for better visualization

### Data Flow

1. Precomputed predictions (`.pkl` files in `predictions/` directory) are generated by an external VGGT model
2. `viser.py` loads the pickle file containing predictions
3. Optional sky segmentation is applied using ONNX runtime (downloads `skyseg.onnx` from HuggingFace)
4. Point cloud and camera frustums are rendered in viser with interactive controls

### Dependencies

Key dependencies visible in the code:
- **viser** - 3D visualization framework
- **numpy** - Numerical operations
- **torch** - PyTorch (for loading predictions)
- **onnxruntime** - For sky segmentation (optional)
- **opencv (cv2)** - Image processing
- **tqdm** - Progress bars

## Common Commands

### Run visualization server

```bash
# Basic usage with default predictions file
python viser.py

# Load specific predictions file
python viser.py --predictions_path predictions/vggt_predictions_0.pkl

# Use point map mode instead of depth unprojection
python viser.py --use_point_map

# Enable sky segmentation (requires onnxruntime)
python viser.py --mask_sky --image_folder /path/to/images

# Change confidence threshold (percentage of low-confidence points to filter)
python viser.py --conf_threshold 25.0

# Use different port
python viser.py --port 8081

# Run in background mode (non-blocking)
python viser.py --background_mode
```

### View all available options

```bash
python viser.py --help
```

## Visualization Features

When running the viser server, the following GUI controls are available:

- **Confidence Percent** slider (0-100) - Filters out the specified percentage of lowest confidence points
- **Show Points from Frames** dropdown - View points from all frames or individual frames
- **Show Cameras** checkbox - Toggle visibility of camera frames and frustums

Click on camera frustums to move the viewpoint to that camera's position.

## File Structure

- `viser.py` - Main visualization script
- `predictions/` - Directory containing precomputed VGGT predictions (`.pkl` files)
- `skyseg.onnx` - ONNX model for sky segmentation (downloaded on first use)
- `README.md` - Project description
- `.gitignore` - Excludes `.pt` and `.pkl` files from git

## Notes

- The repository relies on externally generated prediction files; there is no training or inference code in this repo
- Sky segmentation downloads `skyseg.onnx` from: https://huggingface.co/JianyuanWang/skyseg/resolve/main/skyseg.onnx
- Point clouds are automatically centered around the scene center for better visualization
- The visualization supports both "world_points" mode and depth-based unprojection mode
